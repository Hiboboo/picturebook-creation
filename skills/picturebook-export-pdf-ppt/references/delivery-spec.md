# 绘本交付规格

## 页面结构

固定顺序：

1. 封面：使用 `picturebook-story-design` 已生成的封面完整图片。
2. 故事内页：按故事页码逐页放入已生成的完整图片，每页一张，顺序不能跳页、乱序或按文件修改时间排序。
3. 家长页：故事寓意、亲子互动、适合年龄。

## 图片整理规则

- 导出阶段不生成新图片，不重绘、不换图、不扩图；只引用故事设计阶段已经生成的完整图片。
- 图片顺序优先来自故事包素材清单；如果素材清单缺失，按文件名页码自然排序：`cover` 在前，`page-01`、`page-02`、`page-03` 依次排列。
- 同一页存在预览图和最终图时，优先使用最终图；只有没有最终图时才使用 preview 图。
- 每张图片必须完整放入对应页面，保持原图比例；允许等比缩放和留边，不允许裁掉书名、署名、角色脸部、故事文字或关键动作。
- 如果故事文字已经嵌入图片，默认不要重复叠加文本；需要可编辑文本层时，覆盖层必须与图片中文字一致且不能遮挡画面。

## 版式

- 页面比例优先匹配 `picturebook-story-design` 已生成图片的比例；若图片为 `3:4` 竖版，PDF/PPT 页面也优先使用 `3:4` 竖版。
- 正文使用大字号、高对比、圆润字体；不要使用拥挤段落。
- 文字区必须避开角色脸部、主要动作和画面焦点。
- 同一故事中页边距、标题、页码、署名位置保持一致。
- 封面署名固定为 `图/稳：小葫芦`。

## 文件命名

使用可读文件名：

- `<series-slug>-<title-slug>.pdf`
- `<series-slug>-<title-slug>.pptx`

如果中文 slug 不稳定，可保留中文标题并移除文件系统不安全字符。

## 验证

交付前必须检查：

- PDF 文件存在且大小不为 0。
- PPTX 文件存在且大小不为 0。
- 页数等于 `1 + 故事页数 + 1`。
- 封面有书名和固定署名 `图/稳：小葫芦`。
- 所有封面和内页图片来自 `picturebook-story-design` 已生成素材，没有在导出阶段重新生成图片。
- 每个故事页都有对应图片，且图片顺序与故事页码一致。
- 每个故事页都有对应故事文字；如果文字已在图片中，不重复叠加造成重影。
- 没有成人、暴力、性、恐怖、危险模仿等幼儿不适内容。
