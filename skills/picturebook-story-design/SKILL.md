# 幼儿绘本故事设计

## 必须先做

如果用户提供了绘本系列、主题、名字、书名或故事概念，先使用 `picturebook-check-local-record`。没有完成本地查重前，不要开始构思故事。

创作前读取：

- `references/child-safety-and-consistency.md`
- `references/story-output-contract.md`

## 文字创作原则

- 面向 `2-6` 岁幼儿读物：温柔、清晰、低冲突、可理解，让孩子看得懂、大人读得顺、翻完还想再读。
- 从儿童视角出发，使用孩子熟悉的生活经验和情绪入口，如吃饭、睡觉、下雨、朋友、害怕、分享、等待、分离、亲子陪伴；不要站在成人角度讲大道理。
- 一本故事只保留一个核心主题，如勇敢尝试、认识春天、学会等待、妈妈的爱；避免多主题、长时间线、多重转折或复杂人物关系。
- 情绪价值优先于知识灌输，让孩子感到安全、温暖、有趣、被理解；知识内容必须故事化、画面化、体验化。
- 每页只承载一个清楚动作或情绪；正文短、口语化、有节奏，优先一句主文本，适合朗读和复述。
- 使用可预测的重复结构：可重复一个句式、动作、问题、拟声词或韵律，让孩子获得安全感和参与感。
- 页与页之间必须有推进：每页都要出现新的发现、角色、情绪变化、动作结果或环境变化，给孩子“翻下一页”的动力。
- 结尾温暖、有满足感，不用直接说教；用一个小小的拥抱、完成、发现、合作或成长，让孩子安心、开心、被鼓励。
- 避免文字太多、成人说教、百科式知识密度和“文字写什么，画面只重复画什么”的浪费。文字可以说动作，画面要补充情绪、环境、关系和隐藏细节。
- 故事正文文字必须直接融入绘本图片构图中，不要为了写文字单独留出一整片空白区域，也绝对不要给文字创造固定背板、固定卡片或固定标签框。默认把文字放在左上 `1/4` 或右上 `1/4` 版面内，与背景自然融合；特殊画幅场景可按画面重心调整位置，但必须保持清晰可读、不遮挡角色脸部和关键动作。最终 PDF/PPT 可用可编辑文本覆盖层精确排版。
- 故事和文字中严禁出现成人才能看的内容或儿童不宜内容，包括暴力、血腥、恐吓、性暗示、裸露、危险模仿、歧视、羞辱、酒精烟草、药物、武器、怪诞恐怖元素。成人角色可以出现，但必须是温和、守护性、幼儿读物适宜的形象。

## 构图创作原则

- 封面和所有内页图片统一使用 `3:4` 竖版比例。
- 受欢迎的低龄绘本通常先读到书名，再读到署名，再看主体图。封面书名要明显大、稳、易读，成为第一视觉层级；图/文署名行固定使用 `图/文：小葫芦`，并使用普通字体、黑色或深灰、小字号、规整排版，看起来像人工编辑的普通图书版式，不要使用花哨特效、AI感装饰字或过度艺术化字体。
- 每一幅画面先确定“本页中心意思”，再从专业导演角度设计构图：光线、视角、景别、背景元素、画幅比例、颜色、氛围、人物调度都必须服务这个中心意思。
- 连续场景必须先建立“空间连续性地图”：起点、目标点、行动路线、道路/桥/房间等路径轴线、关键地标、角色行进方向、镜头主侧和视线方向。后续页面只能在这张地图上推进，不能每页重新发明场景。
- 涉及“从 A 到 B”的故事时，目标点必须前后一致，并随页间推进逐渐接近或进入。若起点是路口一侧、目标是过街后的公园门口，公园门口必须稳定出现在同一目标侧，不能忽近忽远、忽左忽右或消失成无关背景。
- 斑马线、道路边缘、河岸、走廊、门口、桥面等是空间骨架。它们的透视方向、起止关系和连接对象必须稳定对应起点与目标点；除非故事明确需要转弯，否则不要让路径轴线歪斜、断开或指向错误目标。
- 角色视角和镜头方向要有连续性。默认保持同一主视角/同一越轴侧来讲述行动；如果需要切换为反打、俯视、近景或角色第一视角，必须说明切换目的，并保留可识别的地标、交通灯、门口、树、店铺等空间锚点，避免读者迷路。
- 画面必须能独立讲故事。即使不读文字，孩子也应能通过连续动作、表情、关系、环境变化和细节线索大致看懂发生了什么。
- 构图不能太单一，也不能为了显得丰富而堆满细节。优先保持主体清楚、元素规整、层次明确：每页只安排服务中心意思的必要角色、道具和环境线索，允许大面积干净留白、纯色背景或简单场景；避免过密花草、气泡、高光、纹理、装饰物和无关小物让画面一眼显得像 AI 生成。
- 画幅视角适当放大，优先使用更开阔的中景、远中景或轻微俯视/平视镜头，容纳角色互动、家庭氛围、环境线索和故事关系；避免过度贴脸、局促或裁切过紧。
- 光线要表达情绪：温暖陪伴可用柔和晨光/灯光，探索发现可用明亮侧光，安静等待可用低对比柔光；不要使用阴森、压迫、惊悚的光影。
- 背景元素要参与叙事，而不是填空装饰。用窗外天气、桌面物件、玩具位置、植物状态、家庭成员的小动作等暗示时间、情绪和页间变化；不需要的细节宁可删掉，保持画面像人工挑选过的绘本元素。
- 色彩搭配必须根据故事场景、内容、环境和情绪精准设计。先确定主色、辅助色和少量强调色：亲子陪伴可用温暖低饱和色，舞台/形状/颜色主题可用更鲜明的蜡笔色，夏天可用清亮绿黄蓝，夜晚/睡前可用柔和低对比色；不要套用固定粉彩、固定高饱和或随机彩虹色。
- 颜色和氛围要随情绪轻微推进：开头可清新稳定，中段可更明亮或更有动感，结尾回到温暖满足；保持色板一致，但避免每页同一种色块和同一种构图。
- 画面默认不只出现一个人物。除非当页故事镜头明确只允许单人入镜，否则以温馨家庭为基调安排人物出场，可包含幼儿、父母、祖辈、兄弟姐妹、宠物或陪伴角色；成人角色必须温和、守护性、儿童适宜。
- 人物、形象、服饰、环境、色彩、叙事口吻必须前后一致；变化来自动作、情绪、镜头和场景细节，而不是随意改变角色设定。
- 文字位置也是构图的一部分。默认在左上或右上 `1/4` 的低细节区域自然融入，必要时用天空、墙面、窗帘、地面光影、树叶间隙等背景承接文字；不得使用固定背板、卡片、标签框。

## 必需输出

按 `story-output-contract.md` 的结构输出完整故事包，至少包含：

- 输出图片：故事书封面。封面必须包含大字号书名和固定署名行 `图/文：小葫芦`。书名要比署名明显更大、更醒目；署名行使用普通字体、黑色或深灰、小字号，靠近书名下方或右下方并保持舒适间距，像人工编辑排版。
- 输出图片：每一页故事画面构图。每页构图必须包含本页故事文字的位置、排版方式和最终嵌入方案；若用户要求内页署名，只使用 `图/文：小葫芦`，且必须小而普通，不抢正文和主画面。
- 输出文字：这个故事想告诉孩子什么。
- 输出文字：亲子互动。
- 输出文字：适合幼儿年龄。

如果当前环境可用 `imagegen`，为封面和每页生成实际图片。若图片数量较多，先生成封面和 1 页样张，确认风格后再批量生成其余页面。

## 一致性步骤

1. 建立“故事设定卡”：系列、主题、书名、年龄、核心主题、儿童视角、目标情绪、重复结构、风格、主角、配角、场景、色板、镜头语言、空间连续性地图。
2. 建立“角色设定卡”：每个幼儿/动物/拟人角色的外形、服装、表情范围、比例、禁改项。
3. 写页表：封面 + 每页 `本页中心意思 / 故事文字 / 文字创作说明 / 页间推进 / 空间连续性校验 / 导演式构图 / 文字融入位置 / 图像提示词 / 安全负面提示词`。
4. 自检安全、文字原则、构图原则和一致性，发现违规内容时直接改写，不把违规版本展示给用户。
5. 输出最终故事包，并提示可继续使用 `picturebook-export-pdf-ppt` 生成 PDF/PPT。

## 自检清单

- 是否已查重并登记草稿。
- 是否没有成人才能看的内容或儿童不宜内容。
- 是否没有暴力、性、恐怖、危险模仿等幼儿不适内容。
- 是否从儿童视角出发，主题单一清晰，语言短、口语化、有节奏。
- 是否有重复结构、页间推进和温暖满足的结尾。
- 是否避免成人说教、文字太多、剧情过复杂、知识密度过高。
- 是否每页主角外观一致。
- 是否每页画面围绕中心意思设计光线、视角、背景元素、画幅比例、颜色和氛围。
- 是否连续场景先建立空间连续性地图，并保持起点、目标点、路线轴线、关键地标、角色视角和镜头方向前后一致。
- 是否斑马线、道路、门口、桥面等空间骨架稳定连接故事的起点和目标点，没有歪斜、漂移或指向错误目标。
- 是否每页画面细节密度适中、规整清楚，没有为了“丰富”而堆满花草、纹理、气泡、高光或无关道具。
- 是否每页色彩方案根据故事场景、内容、环境和情绪精准设计，而不是套用固定粉彩、高饱和或随机彩虹色。
- 是否除必要的单人镜头外，不只出现一个人物，并保持温馨家庭基调。
- 是否封面包含大字号书名和固定署名行 `图/文：小葫芦`，署名使用普通字体、小字号、黑色或深灰，像人工编辑排版。
- 是否每页都给出本页故事文字及其嵌入位置，默认位于左上或右上 `1/4` 版面内，且没有固定背板、卡片或标签框。
- 是否有“想告诉孩子什么”“亲子互动”“适合幼儿年龄”。
